Zabbix FinOps Export

คู่มือใช้งาน Script ดึง Trend เพื่อทำ Right-size

ใช้สำหรับดึงข้อมูล CPU, Memory, Disk, Network และ capacity จาก Zabbix API แล้วออกเป็น CSV สำหรับวิเคราะห์ VM sizing และ FinOps

ภาพรวม ติดตั้ง ตั้งค่า วิธีรัน ตัวอย่างข้อมูล แก้ปัญหา

ภาพรวมไฟล์ที่ต้องรู้

run_zabbix_finops.py ตัวหลักที่แนะนำให้ใช้ อ่านค่าจาก zabbix_finops_config.json แล้วสั่ง export ให้
run_finops_export_batches.py ใช้แบ่ง host เป็น batch และแบ่งช่วงข้อมูลย้อนหลังเป็นรายเดือน เหมาะกับ 44 VM / 6 เดือน
zabbix_trend_rightsize.py ตัวดึงข้อมูล trend จาก Zabbix API จริง ใช้ตรง ๆ ได้ถ้าต้องการ export รอบเดียว

Flow การทำงาน

1
ใส่ URL และ token ใน config
2
เลือก group หรือ hosts.txt
3
รัน dry-run เพื่อตรวจ job
4
รัน export จริง
5
เปิด CSV ไปวิเคราะห์ right-size

1. ติดตั้งก่อนใช้งาน

เปิด PowerShell ที่โฟลเดอร์ project: E:\zabbix-export

cd E:\zabbix-export
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt

ถ้าเคยติดตั้งแล้ว ครั้งถัดไปใช้แค่:

cd E:\zabbix-export
.\.venv\Scripts\Activate.ps1

2. สร้างและแก้ config

คัดลอก config ตัวอย่างเป็นไฟล์ใช้งานจริง:

Copy-Item .\zabbix_finops_config.example.json .\zabbix_finops_config.json
notepad .\zabbix_finops_config.json

ตัวอย่าง config สำหรับดึงทั้ง group

{
  "zabbix_url": "https://zabbix.example.com/",
  "zabbix_token": "ใส่-token-จริงตรงนี้",
  "host_group": "Linux servers",
  "hosts_file": "",
  "months": 6,
  "month": "",
  "days": 30,
  "host_batch_size": 10,
  "output_dir": "exports_finops",
  "timestamp_output_dir": true,
  "summary_output": "finops_summary.csv",
  "wide_output": "finops_wide.csv",
  "raw": false,
  "resume": true,
  "completed_months": false,
  "timeout": 120,
  "min_samples": 500
}

ตัวอย่าง config สำหรับเลือกเฉพาะ VM ใน hosts.txt

เหมาะกับกรณีมีรายชื่อ 44 VM ที่ต้องการดึงเท่านั้น

Copy-Item .\hosts.example.txt .\hosts.txt
notepad .\hosts.txt

ตัวอย่าง hosts.txt:

app-prod-01
app-prod-02
db-prod-01
finops-tools-dev

จากนั้นแก้ใน config:

"hosts_file": "hosts.txt",
"host_group": ""
ถ้าใช้ hosts_file script จะดึงเฉพาะ host ที่อยู่ในไฟล์นั้น ถ้าไม่ใช้ ให้ปล่อยเป็นค่าว่างแล้วใช้ host_group แทน

3. วิธีรันที่แนะนำ

เช็คก่อนว่าจะแบ่ง job ยังไง

python .\run_zabbix_finops.py --dry-run

ตัวอย่างผลลัพธ์ที่ควรเห็น:

hosts=44 host_batches=5 months=6 jobs=30
dry-run 2025-12 batch01 hosts=10
dry-run 2025-12 batch02 hosts=10
dry-run 2025-12 batch03 hosts=10
dry-run 2025-12 batch04 hosts=10
dry-run 2025-12 batch05 hosts=4

รันจริงแบบ 6 เดือน แบ่งรายเดือนและ batch host

python .\run_zabbix_finops.py

ค่า default จะสร้างโฟลเดอร์ใหม่ตามเวลาที่รัน เช่น exports_finops_20260526_003012 เพื่อไม่ให้ export รอบใหม่ทับรอบเก่า

เลือกดึงเต็มเดือนที่ต้องการ

ใช้ --month YYYY-MM ได้เลย script จะหา last day ของเดือนให้อัตโนมัติ ไม่ว่าจะเป็น 28, 29, 30 หรือ 31 วัน

python .\run_zabbix_finops.py --month 2026-02 --dry-run
python .\run_zabbix_finops.py --month 2026-02

ถ้าต้องการหลายเดือนให้ใส่ซ้ำ:

python .\run_zabbix_finops.py --month 2026-01 --month 2026-02 --month 2026-03

หรือใส่ใน config:

"month": "2026-02"

ไฟล์ที่ได้หลัก ๆ คือ:

  • exports_finops_YYYYMMDD_HHMMSS\finops_wide.csv ข้อมูล wide format หนึ่งแถวต่อ VM ต่อ timestamp ใช้กับ Excel/Power BI ได้ง่าย
  • exports_finops_YYYYMMDD_HHMMSS\finops_summary.csv summary แบบ metric-per-row มี p95/p99/signal สำหรับวิเคราะห์เพิ่ม
  • exports_finops_YYYYMMDD_HHMMSS\YYYY-MM\ ไฟล์แยกรายเดือนและ batch

รันแบบเร็ว ดึง 30 วันครั้งเดียว

python .\run_zabbix_finops.py --mode single

กรณีอยากรัน exporter ตรง ๆ

$env:ZABBIX_URL = "https://zabbix.example.com/"
$env:ZABBIX_TOKEN = "ใส่-token-จริงตรงนี้"

python .\zabbix_trend_rightsize.py `
  --group "Linux servers" `
  --days 30 `
  --output rightsize.csv `
  --wide-output rightsize_wide.csv

ความหมายคอลัมน์สำคัญ

คอลัมน์ ความหมาย ใช้ดูอะไร
cpu_min/cpu_max/cpu_avg CPU utilization ต่อชั่วโมงเป็นเปอร์เซ็นต์ หา VM ที่ CPU ต่ำต่อเนื่องหรือ peak สูงผิดปกติ
memory_min/memory_max/memory_avg Memory used ต่อชั่วโมงเป็นเปอร์เซ็นต์ เช็คว่าลด RAM ได้ไหม หรือมี risk memory เต็ม
disk_min/disk_max/disk_avg Disk used ต่อชั่วโมงเป็นเปอร์เซ็นต์ ดูพื้นที่ disk และ growth pattern
network_in/network_out Traffic เฉลี่ยต่อชั่วโมงจาก Zabbix trend ดู workload ที่ network ต่ำมากหรือมี traffic สูง
vcpu/memory_total_gb/disk_total_gb Capacity ที่ดึงจาก Zabbix item ล่าสุด ใช้เทียบว่าควรลดจาก size เดิมลงแค่ไหน

ตัวอย่างอ่านผลเบื้องต้น

  • Candidate CPU avg ต่ำ, CPU max ต่ำ, memory avg ต่ำ และ network ต่ำต่อเนื่อง มีโอกาส downsize
  • ระวัง CPU avg ต่ำแต่ CPU max สูง แปลว่าอาจมี peak เป็นช่วง ๆ ควรดู summary p95/p99 เพิ่ม
  • ระวัง Memory max สูงกว่า 80-85% ไม่ควรลด RAM ง่าย ๆ
  • ระวัง Disk used สูงหรือโตเร็ว ไม่เกี่ยวกับลด CPU/RAM โดยตรง แต่มี risk storage เต็ม

5. ปัญหาที่เจอบ่อย

ไม่มีไฟล์ zabbix_finops_config.json

Copy-Item .\zabbix_finops_config.example.json .\zabbix_finops_config.json

IP ว่างใน output

แปลว่า host นั้นไม่มี interface ใน Zabbix host object หรือใช้ active check/discovery แบบไม่ได้ผูก interface ต้องเพิ่ม interface ใน Zabbix หรือ map IP จาก CMDB/cloud inventory เพิ่ม

ข้อมูลน้อยและขึ้น insufficient_data

Zabbix มี trend data ไม่พอในช่วงที่ดึง หรือ retention สั้นเกินไป สำหรับ 6 เดือนแนะนำให้มีหลายร้อย hourly samples ต่อ VM

อยากดูว่ามี job อะไรจะถูกรัน แต่ยังไม่อยากยิง API หนัก

python .\run_zabbix_finops.py --dry-run

อยากรันต่อจากรอบที่เคยล้ม

ใน config ให้ตั้ง "resume": true ไว้ script จะ skip batch file ที่มีอยู่แล้วและไม่ว่าง
